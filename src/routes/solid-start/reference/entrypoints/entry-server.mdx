title: entry-server.tsx
---

`entry-server.tsx` is where your app starts on the server.  It does so by passing in our `<StartServer>` to a [`render`](http://localhost:3000/reference/rendering/render) function. `<StartServer>` takes a document component which will serve as the static document for your application.

```tsx twoslash
import { createHandler, StartServer } from "@solidjs/start/server";

export default createHandler(() => (
  <StartServer
    document={({ assets, children, scripts }) => (
      <html lang="en">
        <head>
          <meta charset="utf-8" />
          <meta name="viewport" content="width=device-width, initial-scale=1" />
          <link rel="icon" href="/favicon.ico" />
          {assets}
        </head>
        <body>
          <div id="app">{children}</div>
          {scripts}
        </body>
      </html>
    )}
  />
));
```



## Reference

### `createHandler(renderFn, options)`

This calls the underlying Solid render function, and passes the options to it.

| Prop           | Type           | Default   | Description                                                                       |
| -------------- | -------------- | --------- | --------------------------------------------------------------------------------- |
| renderFn       | Function       |           | A function that returns the static document for your application.                 |
| options.mode   | string         | "stream"  | The SSR mode. Options are 'sync', 'async' and 'stream'.                           |


### `<StartServer />`

Component that wraps our application root.

| Prop      | Type     | Description                                                                                   |
| --------- | -------- | --------------------------------------------------------------------------------------------- |
| document  | Function | A function that returns the static document for your application.                             |
